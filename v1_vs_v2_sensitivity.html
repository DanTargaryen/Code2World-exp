Code2world — v1 vs v2 Code 敏感性对比

v1 = 9 动作(含向下) · v2 = 6 动作(去掉3个向下动作: 左/左上/noop/上跳/右/右上) · 其余完全相同

核心结论：受限动作集对"整体命中率"无提升(都是 57%, 4/7)，但在多个细分指标上改善——motion 方向正确性 3/6→5/6，code 消融重要性 1.80×→1.89×。v1 最弱的 base 有改善,但 fast/highjump 反而退步。net 来看是"换了短板,没有全面提升"。

4. 解读

✅ 受限动作带来的改善

  • motion 方向 3/6→5/6:去掉向下动作后,玩家更多做"水平移动+跳跃",fast/slow 的速度差异、重力差异更容易在连贯运动中体现,方向判断更准
  • code 消融 1.80×→1.89×:code 对预测更重要了,说明受限动作让物理参数的影响更突出
  • 5/7 变体 margin 上升:slow/lowgrav/lowjump/base/(highgrav仍命中)整体区分度提升

⚠️ 未解决 / 退步

  • 命中率仍 57%:base/fast/highjump 三个仍未命中(和 v1 一样 3 个未命中,只是 base 换出、fast 没改善)
  • highjump 仍是老大难:跳跃高度差异只在起跳瞬间体现,即使动作集偏向跳跃,糊的生成仍抓不住瞬时差异。motion 显示 highjump 生成运动量最低(0.011),模型没学会"跳更高"
  • fast 退步:可能因为去掉向下动作后,base/fast 在纯水平移动时反而更像(都只能左右走),速度差异被其他变体的更明显特征盖过
  • token 打乱仍≈1.0:模型把 code 当词袋的本质没变——这是架构层面的问题,不是动作集能解决的
结论：受限动作集是个小幅净正向的改进——它让"持续性物理差异"(速度/重力)更易学,motion 方向和 code 重要性都提升了。但它治标不治本:① highjump 的"瞬时差异"问题要靠解决糊(flow matching)和加密跳跃信号;② "code 当词袋"是架构问题,要改 code 注入方式。下一步优先级应是 Stage 2 flow matching 而非继续调动作集。
