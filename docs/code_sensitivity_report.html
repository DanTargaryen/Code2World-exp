Code2world — Code Sensitivity 评估

验证核心命题：改了 code（源码参数），模型生成是否随之改变？· 7 变体配对 eval 集 · 20 episodes

结论：模型确实读了 code，但敏感度部分而非完全。 混淆矩阵对角线命中率 57%（随机基线 14%），code 消融证明 code 携带显著信息（置零后 latent MSE 翻倍）。说明 code condition 起了作用，但区分力在某些物理维度上还不够锐利——这与第一阶段"画面偏糊、MSE 均值化"的局限一致。

1. 混淆矩阵（最有说服力的指标）

构造方法：固定某变体的初始帧+动作，换上变体 v 的 code 做自回归生成，和所有 7 个变体的真值比 MSE。如果模型读 code，用 v 的 code 生成应最像 v 的真值 → 对角线最小。

怎么读：行 = 用哪个 code 生成，列 = 和哪个变体真值比。每行最小值（最亮）应落在对角线才算命中。当前 4/7 命中（slow / lowgrav / highgrav / lowjump 对角线最小）。

未命中的 3 个（base / fast / highjump）都被 lowjump 或 slow 抢走——这几个变体的物理差异在低分辨率 + 糊的生成下视觉区分度小（base 和 lowjump 的差别只在跳跃高度，平地走时几乎一样），所以容易混淆。highgrav 命中且 MSE 最低(0.05)，因为高重力的视觉特征（快速下坠）最明显、最好区分。

2. 反事实可视化：同输入，换 7 种 code

同一个 seed（关卡布局）+ 同一串动作，唯一变量是喂给模型的 coinrun.cpp 源码。7 个变体并排，肉眼看生成是否随 code 改变：

从左到右：base · fast · slow · lowgrav · highgrav · highjump · lowjump（各用自己的 code，同 init+动作）

看点：注意 highgrav 列玩家下坠更快、lowgrav 列更飘。若所有列完全一样 = code 被无视；当前各列有可见差异但幅度偏小，与 57% 命中率吻合。

关键发现：① 置零 code → MSE 翻倍，强证据表明 code 携带模型依赖的有效信息；② 打乱 token 顺序却几乎无影响——说明模型更多是把 code 当作"词袋/全局特征"用（cross-attention 对 token 顺序不敏感），而非精读源码的逐行逻辑。这解释了为什么 code 有作用但区分力有限：它抓住了"这是哪类参数配置"的粗粒度信号，但没精确解析出每个数值。

5. 总评与改进方向

✅ 已证明

  • Code condition 确实影响生成（命中率 57% ≫ 随机 14%，消融 MSE 1.8×）
  • 物理特征明显的变体（highgrav）区分得最好
  • 反事实闭环成立：同 init+动作，换 code，生成会变

⚠️ 局限

  • 3/7 变体被混淆——细微物理差异在糊的生成下区分度不足
  • token 顺序无关 → 模型把 code 当词袋，没精读逐行逻辑/数值
  • motion 代理噪声大，slow 方向错

→ 改进

  • 解决糊（Stage 2 flow matching）：锐利生成会直接提升区分度和命中率
  • 连续参数采样：逼模型真正"读数值"而非记 7 个类别
  • 更强 code 利用：可试 code 不只进 cross-attn，也做位置敏感的编码；或对比学习显式拉开变体表示
  • 更好的物理量评估：从像素检测玩家坐标，比 motion 代理精确
一句话：第一阶段成功证明了 code-conditioned world model 的核心命题成立——模型读了 code 且生成随 code 改变。当前敏感度受限于"画面糊"和"code 被当词袋"，这两点恰好指向 Stage 2（flow matching 解决糊）和数据/架构改进（连续采样 + 强化 code 精读）。
